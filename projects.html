Introduction to Fast Data Structure LTree

Introduction

The chosen data structure is called LTree. LTree grows like a tree and children are stored as a list, hence the name LTree (L→list and Tree). This data structure is inspired by a data structure called VList which means vector of list. Our LTree is similar in some way but makes insertion faster.

GitHub — kaybee1928/Ltree

Summary

LTree has a data part and stores children in the form of a list. Number of elements in children grows exponentially: 1, 2, 4, 8, …. This data structure is useful when we need fast random access along with fast insertion.

Let m = log(n)+log(n/2)+log(n/4)... until log gives 1.
For 2^64 = 18446744073709551616 elements we get m = 64+63+...+1 = 2080

Fast Merge Sort (ARootSort)

ARoot Sort is a highly optimised merge sort which performs two different kinds of merge sort based on degree of sortedness of the input after subarray reversing. It adaptively chooses between natural merge and forced merge depending on how sorted the current partition is.

  • Adaptive strategy based on measured sortedness of subarrays.
  • Subarray reversing as a pre-pass to improve merge efficiency.
  • Benchmarked against standard merge sort — outperforms on partially sorted inputs.
